## 一、模块概述

### 1.1 功能定位
语义查询子模块是量化系统的**智能数据查询层**，允许用户使用自然语言描述查询条件，系统自动解析并转换为SQL查询，返回符合条件的股票数据。

### 1.2 核心价值
- **降低使用门槛**：非技术用户无需了解数据库结构和SQL语法
- **提高查询效率**：一句话完成复杂的多条件组合查询
- **灵活扩展**：支持精确条件、模糊条件、历史数据、组合条件等多种查询模式

### 1.3 技术架构
```
用户输入自然语言查询
    ↓
语义解析器 (SemanticQueryParser)
    ├─ 分词与词性标注
    ├─ 字段识别与映射
    ├─ 条件解析与转换
    └─ 逻辑关系构建
    ↓
SQL生成器 (SQLGenerator)
    ├─ 表选择与JOIN
    ├─ WHERE条件构建
    ├─ ORDER BY排序
    └─ LIMIT限制
    ↓
查询执行器 (QueryExecutor)
    ├─ SQL执行
    ├─ 结果格式化
    └─ 错误处理
    ↓
返回结构化结果
```

### 2.2 语义解析器 (semantic_parser.py)

#### 2.2.1 解析流程
```
输入: "沪深主板,非ST,市盈率,行业,昨日最高涨幅大于4,前2日最低涨幅小于8"
    ↓
步骤1: 分词与词性标注
    ["沪深主板", ",", "非ST", ",", "市盈率", ",", "行业", ",", "昨日最高涨幅", "大于", "4", ",", "前2日最低涨幅", "小于", "8"]
    ↓
步骤2: 字段识别
    {
        "select_fields": ["ts_code", "name", "pe_ttm", "industry"],
        "conditions": [
            {"type": "board", "value": "沪深主板"},
            {"type": "st_status", "value": "非ST"},
            {"type": "field_condition", "field": "昨日最高涨幅", "operator": ">", "value": 4},
            {"type": "field_condition", "field": "前2日最低涨幅", "operator": "<", "value": 8}
        ]
    }
    ↓
步骤3: 条件转换
    WHERE ts_code LIKE ('000%', '001%', ...) 
      AND name NOT LIKE '%ST%'
      AND (SELECT MAX(pct_chg) FROM daily WHERE ts_code = d.ts_code AND trade_date = '{yesterday}') > 4
      AND (SELECT MIN(pct_chg) FROM daily WHERE ts_code = d.ts_code AND trade_date BETWEEN '{date_2d_ago}' AND '{yesterday}') < 8
```

## 三、查询模式详解

### 3.1 智能匹配数据字段值查询

**示例**: `query=股票代码,股票名称,涨幅,市盈率,行业,量比,买一量,主力流入,昨日开盘涨幅和昨日开盘成交额`

**解析逻辑**:
1. 识别逗号分隔的字段列表
2. 映射到数据库字段
3. 生成SELECT语句
4. 默认返回最新交易日数据

**生成SQL**:
```sql
SELECT 
    sb.ts_code AS 股票代码,
    sb.name AS 股票名称,
    d.pct_chg AS 涨幅,
    db.pe_ttm AS 市盈率,
    sb.industry AS 行业,
    db.volume_ratio AS 量比,
    mf.buy_sm_amount AS 买一量,
    mf.net_mf_amount AS 主力流入,
    (d.open - d.pre_close) / d.pre_close * 100 AS 昨日开盘涨幅,
    d.open * d.vol AS 昨日开盘成交额
FROM stock_basic sb
LEFT JOIN daily d ON sb.ts_code = d.ts_code AND d.trade_date = '20260417'
LEFT JOIN daily_basic db ON sb.ts_code = db.ts_code AND db.trade_date = '20260417'
LEFT JOIN moneyflow mf ON sb.ts_code = mf.ts_code AND mf.trade_date = '20260417'
ORDER BY d.pct_chg DESC
LIMIT 100
```

### 3.2 字段精确条件查询

**示例**: `query=沪深主板,非ST,市盈率,行业,昨日最高涨幅大于4%,前2日最低涨幅小于8%`

**解析逻辑**:
1. 识别板块条件：沪深主板 → ts_code前缀匹配
2. 识别ST状态：非ST → name NOT LIKE '%ST%'
3. 识别数值条件：昨日最高涨幅大于4% → 子查询
4. 识别时间条件：前2日 → 日期范围

**生成SQL**:
```sql
SELECT 
    sb.ts_code,
    sb.name,
    db.pe_ttm AS 市盈率,
    sb.industry AS 行业
FROM stock_basic sb
LEFT JOIN daily_basic db ON sb.ts_code = db.ts_code AND db.trade_date = '20260417'
WHERE (
    sb.ts_code LIKE '000%' OR sb.ts_code LIKE '001%' OR 
    sb.ts_code LIKE '002%' OR sb.ts_code LIKE '600%' OR 
    sb.ts_code LIKE '601%' OR sb.ts_code LIKE '603%' OR 
    sb.ts_code LIKE '605%'
)
AND sb.name NOT LIKE '%ST%'
AND (
    SELECT MAX(d.pct_chg) 
    FROM daily d 
    WHERE d.ts_code = sb.ts_code 
      AND d.trade_date = '20260416'
) > 4
AND (
    SELECT MIN(d.pct_chg) 
    FROM daily d 
    WHERE d.ts_code = sb.ts_code 
      AND d.trade_date BETWEEN '20260414' AND '20260415'
) < 8
LIMIT 100
```

### 3.3 字段模糊条件查询

**示例**: `query=集合竞价抢筹,30日均线向上,macd底背离`

**解析逻辑**:
1. 匹配模糊条件词典
2. 转换为具体SQL条件
3. 组合多个条件

**生成SQL**:
```sql
SELECT 
    sb.ts_code,
    sb.name,
    d.pct_chg,
    db.pe_ttm
FROM stock_basic sb
LEFT JOIN daily d ON sb.ts_code = d.ts_code AND d.trade_date = '20260417'
LEFT JOIN daily_basic db ON sb.ts_code = db.ts_code AND db.trade_date = '20260417'
LEFT JOIN stk_factor_pro fp ON sb.ts_code = fp.ts_code AND fp.trade_date = '20260417'
WHERE d.volume_ratio > 2 
  AND d.open > d.pre_close * 1.01
  AND fp.macd_dif > fp.macd_dif_prev
  AND d.close < d.close_prev
LIMIT 100
```

### 3.4 多个指定日期条件查询

**示例**: `query=2015-09-13跌停,2015-09-14涨停`

**解析逻辑**:
1. 识别日期格式：YYYY-MM-DD
2. 转换为YYYYMMDD格式
3. 为每个日期生成独立条件
4. 使用AND连接

**生成SQL**:
```sql
SELECT 
    sb.ts_code,
    sb.name,
    d1.pct_chg AS '2015-09-13涨幅',
    d2.pct_chg AS '2015-09-14涨幅'
FROM stock_basic sb
INNER JOIN daily d1 ON sb.ts_code = d1.ts_code AND d1.trade_date = '20150913'
INNER JOIN daily d2 ON sb.ts_code = d2.ts_code AND d2.trade_date = '20150914'
WHERE d1.pct_chg <= -9.8
  AND d2.pct_chg >= 9.8
LIMIT 100
```

---

### 3.5 多个条件组合查询

**示例**: `query=主板,市盈率大于60,或者(华为概念并且市盈率小于50)`

**解析逻辑**:
1. 识别逻辑操作符：并且→AND，或者→OR
2. 识别括号分组
3. 构建复合条件

**生成SQL**:
```sql
SELECT 
    sb.ts_code,
    sb.name,
    db.pe_ttm AS 市盈率,
    sb.industry AS 行业
FROM stock_basic sb
LEFT JOIN daily_basic db ON sb.ts_code = db.ts_code AND db.trade_date = '20260417'
WHERE (
    (
        (sb.ts_code LIKE '60%' OR sb.ts_code LIKE '00%')
        AND db.pe_ttm > 60
    )
    OR (
        sb.industry LIKE '%华为%'
        AND db.pe_ttm < 50
    )
)
LIMIT 100
```

## 五、错误处理与边界情况

### 5.1 常见错误类型

| 错误类型 | 示例 | 处理方式 |
|---------|------|---------|
| 字段不存在 | "查询股票代码,不存在的字段" | 返回错误提示，列出可用字段 |
| 日期格式错误 | "2015/09/13涨停" | 自动转换为YYYY-MM-DD格式 |
| 日期超出范围 | "2010-01-01涨停"（数据从2015年开始） | 提示数据起始日期 |
| 逻辑语法错误 | "主板并且或者市盈率大于60" | 返回语法错误提示 |
| 无匹配结果 | "查询科创板涨停股票"（当日无） | 返回空结果，提示无匹配 |

### 5.2 错误响应格式

```json
{
    "success": false,
    "error": {
        "type": "field_not_found",
        "message": "字段'不存在的字段'不存在",
        "suggestion": "可用字段包括：股票代码、股票名称、涨幅、市盈率...",
        "query": "原始查询语句"
    }
}
```

## 六、性能优化策略

1. **索引使用**
   - ts_code + trade_date 复合索引
   - 常用查询字段建立索引

2. **查询缓存**
   - 相同查询条件缓存结果
   - 缓存有效期：交易日当天

3. **分页查询**
   - 默认返回100条
   - 支持offset/limit分页

### 6.2 并发控制

1. **连接池管理**
   - DuckDB只读连接池
   - 最大并发连接数限制

2. **查询超时**
   - 设置查询超时时间（默认30秒）
