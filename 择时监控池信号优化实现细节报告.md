## 概述

本报告详细分析了当前择时监控池（Pool 1）信号逻辑的优化方案，提出"日线筛选→盘中确认"的两阶段信号处理框架，旨在提高信号的质量和稳定性。

## 当前信号逻辑分析

### 现有问题
1. **信号触发简单叠加**：当前实现是日线因子与盘中数据的简单叠加，缺乏严谨的筛选机制
2. **缺乏宏观判断**：未充分利用筹码集中度、压力支撑位等宏观信息
3. **微观结构利用不足**：tick、五档盘口、逐笔数据的深度分析缺失
4. **风险控制薄弱**：缺乏基于日线分析的动态止损止盈机制

## 优化方案设计

### 1. 基于专业优化建议的深度集成方案

**核心优化方向**：提高中短期趋势判断准确性，增强信号持续性和可靠性

#### 第一阶段：因子优化与增强（深度集成）
**目标**：优先实现中短期趋势判断相关因子，增强信号宏观基础
**集成方式**：在日线筛选条件中集成事件驱动、资金流、趋势动量因子
**数据源**：现有 `stk_factor_pro` + 新增 `EV_*`、`MF_*`、`M_*` 因子

#### 第二阶段：多周期共振确认机制
**目标**：通过多时间周期共振提高信号可靠性
**集成方式**：在现有信号检测基础上增加60分钟线确认条件
**数据源**：日线 + 60分钟线技术指标数据

#### 第三阶段：动态阈值与市场环境适应
**目标**：根据市场波动率动态调整信号阈值
**集成方式**：基于ATR、历史波动率动态调整BOLL、RSI阈值
**数据源**：历史波动率数据 + 实时市场环境指标

#### 第四阶段：信号持续性评估与衰减机制
**目标**：避免"一闪而过"的无效信号，提供更长的决策时间
**集成方式**：信号强度随时间衰减，特定条件下失效
**数据源**：信号触发时间戳 + 关键支撑位变化

#### 第五阶段：中长期市场结构分析
**目标**：侧重中长期资金行为分析，识别主力资金动向
**集成方式**：集成大宗交易、机构调研、融资融券数据
**数据源**：大宗交易数据 + 机构持仓数据 + 融资融券数据

#### 第六阶段：专业清仓止盈信号体系
**目标**：提供基于技术分析的止损止盈建议
**集成方式**：多维度综合判断，分级止盈建议
**数据源**：技术指标 + 价格目标 + 量价关系 + 市场结构

```python
class FactorOrthogonalization:
    """因子正交化处理 - 解决多重共线性问题"""
    
    def __init__(self):
        # 因子分类映射
        self.factor_categories = {
            'momentum': ['rsi6', 'rsi12', 'cci', 'roc', 'williams_r'],
            'trend': ['macd', 'macd_signal', 'ma5', 'ma10', 'ma20', 'adx'],
            'volatility': ['boll_upper', 'boll_mid', 'boll_lower', 'atr', 'std_dev'],
            'volume': ['volume_ratio', 'volume_ma5', 'volume_ma20', 'turnover_rate'],
            'price_level': ['price', 'close', 'high', 'low']
        }
    
    def orthogonalize_score(self, factors_dict):
        """因子正交化评分 - 同类因子取最大值，跨类因子相加"""
        
        category_scores = {}
        
        # 按类别计算最大得分
        for category, factor_list in self.factor_categories.items():
            category_max = 0
            for factor in factor_list:
                if factor in factors_dict:
                    # 使用条件概率模型计算因子得分
                    factor_score = self.calculate_conditional_probability_score(factor, factors_dict[factor])
                    category_max = max(category_max, factor_score)
            category_scores[category] = category_max
        
        # 跨类别因子相加（避免多重共线性）
        total_score = sum(category_scores.values())
        
        return {
            'total_score': min(100, total_score),
            'category_scores': category_scores,
            'orthogonalized': True
        }
    
    def calculate_conditional_probability_score(self, factor_name, factor_value):
        """基于条件概率的因子评分 - 避免硬编码分值"""
        
        # 从历史数据中获取该因子的胜率
        win_probability = self.get_historical_win_probability(factor_name, factor_value)
        
        # 使用胜率作为评分基础
        if win_probability > 0.7:
            return 25  # 高胜率因子
        elif win_probability > 0.6:
            return 20  # 中等胜率
        elif win_probability > 0.5:
            return 15  # 基础胜率
        else:
            return 10  # 低胜率
    
    def get_historical_win_probability(self, factor_name, factor_value):
        """从历史数据中计算因子条件胜率"""
        
        # 这里需要连接历史数据库，计算P(Win | Factor)
        conn = duckdb.connect(DB_PATH)
        try:
            # 根据因子类型构建查询条件
            if factor_name in ['rsi6', 'rsi12']:
                # RSI超卖条件
                query = f"""
                    SELECT 
                        COUNT(*) as total_cases,
                        SUM(CASE WHEN next_day_return > 0 THEN 1 ELSE 0 END) as win_cases
                    FROM factor_performance 
                    WHERE factor_name = '{factor_name}' 
                    AND factor_value < 30  -- RSI超卖
                """
            elif factor_name in ['boll_lower']:
                # 跌破BOLL下轨条件
                query = f"""
                    SELECT 
                        COUNT(*) as total_cases,
                        SUM(CASE WHEN next_day_return > 0 THEN 1 ELSE 0 END) as win_cases
                    FROM factor_performance 
                    WHERE factor_name = '{factor_name}' 
                    AND price < boll_lower  -- 跌破下轨
                """
            else:
                # 默认查询
                query = f"""
                    SELECT 
                        COUNT(*) as total_cases,
                        SUM(CASE WHEN next_day_return > 0 THEN 1 ELSE 0 END) as win_cases
                    FROM factor_performance 
                    WHERE factor_name = '{factor_name}'
                """
            
            result = conn.execute(query).fetchone()
            if result and result[0] > 0:
                return result[1] / result[0]
            else:
                return 0.5  # 默认胜率
                
        finally:
            conn.close()


def detect_orthogonalized_resonance(ts_code, daily_factors, hourly_factors):
    """基于因子正交化的多周期共振检测"""
    
    orthogonalizer = FactorOrthogonalization()
    
    # 日线因子正交化评分
    daily_result = orthogonalizer.orthogonalize_score(daily_factors)
    
    # 60分钟线因子正交化评分
    hourly_result = orthogonalizer.orthogonalize_score(hourly_factors)
    
    # 共振判断：两个周期都达到阈值
    has_resonance = daily_result['total_score'] >= 60 and hourly_result['total_score'] >= 60
    
    resonance_signals = []
    
    if has_resonance:
        # 计算综合共振强度
        resonance_strength = (daily_result['total_score'] + hourly_result['total_score']) // 2
        
        # 判断共振类型
        resonance_type = determine_resonance_type(daily_result['category_scores'], hourly_result['category_scores'])
        
        resonance_signals.append({
            'type': f'{resonance_type}_resonance',
            'direction': 'buy',

            'priority': 'high',
            'category_breakdown': {
                'daily': daily_result['category_scores'],
                'hourly': hourly_result['category_scores']
            }
        })
    
    return resonance_signals


def determine_resonance_type(daily_categories, hourly_categories):
    """根据因子类别分布判断共振类型"""
    
    # 计算各类别的主导程度
    daily_max_category = max(daily_categories, key=daily_categories.get)
    hourly_max_category = max(hourly_categories, key=hourly_categories.get)
```

```python
class WashTradeDetector:
    """对倒单检测器"""
    
    def detect_wash_trades(self, tick_data):
        """检测对倒交易"""
        transactions = tick_data.get('transactions', [])
        
        if len(transactions) < 10:
            return {'has_wash_trades': False, 'wash_volume': 0, 'suspicious_pairs': []}
        
        suspicious_pairs = []
        total_wash_volume = 0
        
        # 滑动窗口检测
        window_size = 10
        for i in range(len(transactions) - window_size + 1):
            window = transactions[i:i+window_size]
            
            # 在窗口内寻找对倒模式
            wash_volume, pairs = self.find_wash_trades_in_window(window)
            total_wash_volume += wash_volume
            suspicious_pairs.extend(pairs)
        
        return {
            'has_wash_trades': total_wash_volume > 0,
            'wash_volume': total_wash_volume,
            'suspicious_pairs': suspicious_pairs,
            'wash_intensity': total_wash_volume / sum(tx['volume'] for tx in transactions) if transactions else 0
        }
    
    def find_wash_trades_in_window(self, transactions):
        """在时间窗口内检测对倒交易"""
        wash_volume = 0
        pairs = []
        
        # 按时间排序
        sorted_tx = sorted(transactions, key=lambda x: x['timestamp'])
        
        for i in range(len(sorted_tx)):
            for j in range(i+1, len(sorted_tx)):
                tx1 = sorted_tx[i]
                tx2 = sorted_tx[j]
                
                time_diff = tx2['timestamp'] - tx1['timestamp']
                
                # 对倒特征：短时间内、相同金额、相反方向
                if (time_diff <= 2 and  # 2秒内
                    abs(tx1['amount'] - tx2['amount']) < tx1['amount'] * 0.1 and  # 金额相近
                    tx1['direction'] != tx2['direction'] and  # 方向相反
                    tx1['volume'] > 10000):  # 大单
                    
                    wash_volume += tx1['volume']
                    pairs.append({
                        'tx1': tx1,
                        'tx2': tx2,
                        'time_diff': time_diff,
                        'volume': tx1['volume']
                    })
        
        return wash_volume, pairs
```

```python
            if len(scores) >= 10:  # 有足够数据

                    else:
                        # 性能一般，小幅调整
                        new_factor = current_factor * 1.1
                    
                    self.signal_type_factors[signal_type] = min(new_factor, 2.0)  # 上限2.0
                
                elif avg_score > 0.9:
                    # 性能很好，可以稍微降低衰减率
                    current_factor = self.signal_type_factors.get(signal_type, 1.0)
                    new_factor = current_factor * 0.95  # 稍微降低衰减率
                    self.signal_type_factors[signal_type] = max(new_factor, 0.5)  # 下限0.5
```

```python
class RiskControlStateMachine:
    """风险控制状态机 - 避免"资金绞肉机"陷阱"""
    
    def __init__(self):
        self.stock_states = {}  # {ts_code: {state, exit_reason, exit_time, cooldown_until}}
        self.cooldown_config = {
            'take_profit': 1,      # 止盈冷却期：1天
            'normal_stop_loss': 3, # 正常止损冷却期：3天
            'critical_stop_loss': 5, # 关键支撑跌破冷却期：5天
            'liquidity_drain': 7   # 流动性枯竭冷却期：7天
        }
    
    def record_exit(self, ts_code, exit_reason, exit_price, exit_time):
        """记录退出事件并设置冷却期"""
        
        # 确定冷却期长度
        cooldown_days = self.determine_cooldown_period(exit_reason, exit_price)
        cooldown_until = exit_time + timedelta(days=cooldown_days)
        
        # 记录状态
        self.stock_states[ts_code] = {
            'state': 'cooldown',
            'exit_reason': exit_reason,
            'exit_price': exit_price,
            'exit_time': exit_time,
            'cooldown_until': cooldown_until,
            'cooldown_days': cooldown_days,
            'reentry_conditions': self.get_reentry_conditions(exit_reason)
        }
    
    def determine_cooldown_period(self, exit_reason, exit_price):
        """根据退出原因确定冷却期长度"""
        
        if exit_reason == 'take_profit':
            return self.cooldown_config['take_profit']
        
        elif exit_reason == 'normal_stop_loss':
            return self.cooldown_config['normal_stop_loss']
        
        elif exit_reason == 'critical_support_break':
            return self.cooldown_config['critical_stop_loss']
        
        elif exit_reason == 'liquidity_drain':
            return self.cooldown_config['liquidity_drain']
        
        else:
            return 3  # 默认3天
    
    def get_reentry_conditions(self, exit_reason):
        """根据退出原因确定重新入场条件"""
        
        conditions = []
        
        if exit_reason in ['normal_stop_loss', 'critical_support_break']:
            # 止损退出后，要求重新站上MA20才能入场
            conditions.append({
                'type': 'technical',
                'condition': 'price > ma20',
                'description': '价格重新站上20日均线'
            })
        
        if exit_reason == 'liquidity_drain':
            # 流动性枯竭后，要求成交量恢复正常
            conditions.append({
                'type': 'volume',
                'condition': 'volume_ratio > 1.0',
                'description': '成交量恢复到平均水平以上'
            })
        
        # 所有情况都要求冷却期满
        conditions.append({
            'type': 'time',
            'condition': 'cooldown_expired',
            'description': '冷却期已满'
        })
        
        return conditions
    
    def can_reenter(self, ts_code, current_data):
        """检查是否可以重新入场"""
        
        if ts_code not in self.stock_states:
            return True  # 没有冷却期记录
        
        state = self.stock_states[ts_code]
        current_time = datetime.now()
        
        # 检查冷却期是否已满
        if current_time < state['cooldown_until']:
            return False
        
        # 检查重新入场条件是否满足
        for condition in state['reentry_conditions']:
            if not self.check_reentry_condition(condition, current_data):
                return False
        
        # 所有条件满足，清除冷却期状态
        del self.stock_states[ts_code]
        return True
    
    def check_reentry_condition(self, condition, current_data):
        """检查单个重新入场条件"""
        
        if condition['type'] == 'time' and condition['condition'] == 'cooldown_expired':
            return True  # 时间条件已经在can_reenter中检查
        
        elif condition['type'] == 'technical' and condition['condition'] == 'price > ma20':
            return current_data.get('price', 0) > current_data.get('ma20', 0)
        
        elif condition['type'] == 'volume' and condition['condition'] == 'volume_ratio > 1.0':
            return current_data.get('volume_ratio', 0) > 1.0
        
        return True


class ATRDynamicStopLoss:
    """基于ATR的动态止损体系"""
    
    def __init__(self):
        self.atr_multiplier = {
            'high_volatility': 2.0,   # 高波动率股票：2倍ATR
            'medium_volatility': 2.5, # 中等波动率：2.5倍ATR
            'low_volatility': 3.0     # 低波动率：3倍ATR
        }
    
    def calculate_dynamic_stop_loss(self, entry_price, atr_20, volatility_category):
        """计算动态止损位"""
        
        multiplier = self.atr_multiplier.get(volatility_category, 2.5)
        stop_loss_price = entry_price - (multiplier * atr_20)
        
        # 确保止损位不低于最大可接受亏损（如8%）
        max_loss_price = entry_price * 0.92
        stop_loss_price = max(stop_loss_price, max_loss_price)
        
        return {
            'stop_loss_price': stop_loss_price,
            'atr_multiplier': multiplier,
            'max_loss_percent': (entry_price - stop_loss_price) / entry_price * 100,
            'volatility_category': volatility_category
        }
    
    def determine_volatility_category(self, historical_data):
        """根据历史波动率确定股票波动性类别"""
        
        # 计算20日波动率
        if len(historical_data) < 20:
            return 'medium_volatility'
        
        returns = []
        for i in range(1, len(historical_data)):
            prev_close = historical_data[i-1]['close']
            curr_close = historical_data[i]['close']
            if prev_close > 0:
                daily_return = (curr_close - prev_close) / prev_close
                returns.append(daily_return)
        
        if len(returns) == 0:
            return 'medium_volatility'
        
        import numpy as np
        volatility = np.std(returns) * np.sqrt(252)  # 年化波动率
        
        if volatility > 0.4:
            return 'high_volatility'   # 半导体、科技股等
        elif volatility > 0.2:
            return 'medium_volatility' # 一般股票
        else:
            return 'low_volatility'    # 银行、公用事业等
    
    def should_stop_loss(self, current_price, stop_loss_info, market_condition):
        """判断是否应该止损"""
        
        basic_condition = current_price <= stop_loss_info['stop_loss_price']
        
        # 在极端市场条件下，可以放宽止损条件
        if market_condition == 'extreme_bear':
            # 熊市中，允许价格稍微跌破止损位
            tolerance = stop_loss_info['stop_loss_price'] * 0.02  # 2%容忍度
            return current_price <= (stop_loss_info['stop_loss_price'] - tolerance)
        
        return basic_condition
```

class PositionManager:
    """完整的持仓管理框架"""
    
    def __init__(self):
        self.positions = {}  # ts_code -> position_info
        self.signal_history = []
        
    def update_position(self, ts_code, current_data):
        """更新持仓信息"""
        if ts_code not in self.positions:
            return
            
        position = self.positions[ts_code]
        
        # 更新当前价格和收益
        position['current_price'] = current_data['price']
        position['profit_ratio'] = self.calculate_profit_ratio(position)
        position['max_profit'] = max(position.get('max_profit', 0), position['profit_ratio'])
        position['max_drawdown'] = self.calculate_max_drawdown(position)
        
        # 生成止盈止损信号
        take_profit_signals = generate_take_profit_signals(
            current_data, position['entry_price'], 
            current_data['price'], position['daily_analysis']
        )
        
        stop_loss_signals = generate_stop_loss_signals(
            current_data, position['entry_price'],
            current_data['price'], position['daily_analysis']
        )
        
        # 综合评估信号优先级
        final_signals = self.evaluate_signals_priority(take_profit_signals + stop_loss_signals)
        
        return final_signals
    
    def calculate_profit_ratio(self, position):
        """计算收益率"""
        return (position['current_price'] - position['entry_price']) / position['entry_price'] * 100
    
    def calculate_max_drawdown(self, position):
        """计算最大回撤"""
        if position['profit_ratio'] <= 0:
            return abs(position['profit_ratio'])
        else:
            return max(0, position['max_profit'] - position['profit_ratio'])
    
    def evaluate_signals_priority(self, signals):
        """评估信号优先级"""
        priority_map = {'critical': 100, 'high': 80, 'medium': 60, 'low': 40}
        
        # 按优先级排序
        sorted_signals = sorted(signals, key=lambda x: priority_map.get(x['priority'], 0), reverse=True)
        
        # 过滤重复类型信号，保留优先级最高的
        unique_signals = {}
        for signal in sorted_signals:
            if signal['type'] not in unique_signals:
                unique_signals[signal['type']] = signal
        
        return list(unique_signals.values())
    
    def execute_signal(self, ts_code, signal):
        """执行交易信号"""
        if signal['direction'] == 'sell':
            # 清仓操作
            self.close_position(ts_code, signal)
            self.record_signal_execution(ts_code, signal)
        
    def close_position(self, ts_code, signal):
        """清仓操作 - 清仓后继续监控"""
        if ts_code in self.positions:
            position = self.positions.pop(ts_code)
            
            # 记录交易结果
            trade_result = {
                'ts_code': ts_code,
                'entry_price': position['entry_price'],
                'exit_price': position['current_price'],
                'profit_ratio': position['profit_ratio'],
                'holding_days': self.calculate_holding_days(position),
                'signal_type': signal['type'],
                'exit_reason': signal['message'],
                'exit_time': time.time()
            }
            
            self.record_trade_result(trade_result)
            
            # 清仓后，该股票继续留在监控池中
            # 下一轮循环时会继续评估是否具备新的建仓条件
            # 无需从监控池移除，也无需冷却期

```python
class CompleteTradingPipeline:
    """完整的三阶段交易信号流水线"""
    
    def __init__(self):
        self.daily_screener = DailyStockScreener()
        self.intraday_analyzer = IntradayAnalyzer()
        self.position_manager = PositionManager()
        
    async def run_complete_pipeline(self):
        """运行完整的三阶段交易流水线"""
        
        while True:  # 无限循环，形成闭环
            # 第一阶段：日线筛选
            # 注意：清仓后的股票继续留在监控池中参与筛选
            eligible_stocks = await self.daily_screener.screen_stocks()
            
            # 第二阶段：盘中建仓信号确认
            for stock in eligible_stocks:
                # 跳过当前已持仓的股票（避免重复建仓）
                if stock['ts_code'] in self.position_manager.positions:
                    continue
                    
                tick_data = await self.get_real_time_data(stock['ts_code'])
                
                # 盘中确认评分
                confirmation_score = self.intraday_analyzer.confirm_signal(tick_data, stock)
                
                if confirmation_score >= 70:
                    # 生成建仓信号
                    entry_signal = self.generate_entry_signal(stock, tick_data, confirmation_score)
                    
                    # 执行建仓
                    await self.execute_entry(entry_signal)
                    
                    # 记录持仓
                    self.position_manager.add_position(stock, entry_signal)
            
            # 第三阶段：持仓监控与止盈清仓
            await self.monitor_positions()
            
            # 等待下一轮循环
            await asyncio.sleep(60)  # 每分钟运行一次
    
    async def monitor_positions(self):
        """监控持仓并生成止盈清仓信号"""
        for ts_code in list(self.position_manager.positions.keys()):
            current_data = await self.get_real_time_data(ts_code)
            
            # 更新持仓信息
            signals = self.position_manager.update_position(ts_code, current_data)
            
            # 执行止盈清仓信号
            for signal in signals:
                if signal['strength'] >= 70:  # 强度阈值
                    await self.execute_exit(ts_code, signal)
                    self.position_manager.execute_signal(ts_code, signal)
```

```python
def resolve_signal_conflicts(signals):
    """解决信号冲突，确定最终操作"""
    
    # 信号优先级映射
    priority_weights = {
        'critical': 100,  # 强制止损
        'high': 80,       # 重要止盈/止损
        'medium': 60,     # 一般止盈
        'low': 40         # 建议止盈
    }
    
    # 按优先级和强度综合排序
    weighted_signals = []
    for signal in signals:
        weight = priority_weights.get(signal['priority'], 0) + signal['strength']
        weighted_signals.append((weight, signal))
    
    weighted_signals.sort(reverse=True)
    
    # 返回权重最高的信号
    if weighted_signals:
        return weighted_signals[0][1]
    
    return None
```

```python
        self.update_performance_metrics(trade_record)
```

### 三、信号策略逻辑细节优化（重点回答）

结论：**有，而且是当前最应先优化的部分。**

#### 1）左侧抄底 `left_side_buy`

- 当前问题：
  - 条件并集过宽，`RSI<30` 或 `pct_chg<-3` 可单独触发，误报偏高。
- 建议逻辑：
  - **硬触发**：`price <= boll_lower * (1 + near_pct)`，`near_pct` 建议 `0.003~0.005`。
  - **软确认加分**：
    - `rsi6 < oversold`
    - `pct_chg < -3`
    - `bid_ask_ratio > 1.2`（盘口承接）
    - `lure_short`（诱空）加分
  - **反向否决**：
    - `lure_long` 或 `wash_trade` 且无承接时直接降为观察信号（不新触发）。

#### 2）右侧突破 `right_side_breakout`

- 当前问题：
  - 仅判断“当前价格在中轨上方”，缺少“穿越行为”确认。
- 建议逻辑：
  - **硬触发二选一**：
    - `cross_mid_up`: `prev_price <= boll_mid and price > boll_mid`
    - `break_upper_valid`: `price > boll_upper` 且突破幅度 > `0.2%`
  - **软确认加分**：
    - `volume_ratio > threshold(vol)`（随波动率动态）
    - `ma5 > ma10` 或 `ma10` 斜率为正
    - `real_buy` 主力标签加分
  - **反向否决**：
    - `lure_long` + 卖盘压制明显时，降低触发等级。

#### 3）强度评分

- 由“条件数量*固定分”改为“基准分 + 结构化加减分”：
  - `strength = base + confirm_bonus + market_structure_delta - risk_penalty`
  - 建议区间：
    - `base`：50
    - `confirm_bonus`：0~30
    - `market_structure_delta`：-25~+25
    - `risk_penalty`：0~20

#### 3.5）涨跌停磁吸效应过滤（Pool1 专项）

- 当前问题：
  - 当涨跌幅接近涨停/跌停（如 `|pct_chg| > 8.5%`）时，A股“磁吸效应”会导致盘口流动性快速单边化，常规技术指标连续性失真。
- 建议逻辑：
  - **先判定后评分**（优先级高于 BOLL/RSI/MA）：
    - 计算 `Distance_to_Limit`；
    - 若 `< 1.5%`，标记 `limit_magnet=true`。
  - **处理规则**：
    - `left_side_buy`：直接一票否决（默认）。
    - `right_side_breakout`：只保留专门“打板确认”逻辑；未开启打板模块时降级为观察信号。
  - **风控输出**：
    - 在 `details` 中写入：
      - `limit_magnet`
      - `distance_to_up_limit`
      - `distance_to_down_limit`
      - `limit_filter_action`（blocked / observer_only / board_mode）

#### 4）信号生命周期

- 引入三态：
  - `active`：触发后高可信期
  - `decaying`：未再强化，强度随时间下降
  - `expired`：超时或破关键位或被反向信号终止
- 建议默认：
  - `left_side_buy` 半衰期 4 分钟
  - `right_side_breakout` 半衰期 6 分钟
  - 最大存活 15 分钟（可配）

#### 5）冲突解决

- 同时出现买卖信号时不直接并存展示，使用优先级：
  - `critical risk > reverse_t > right_side_breakout > left_side_buy`
- 同方向多个信号保留最强一条，其他合并为 `details.confirmations`。

### 四、数据与接口改造（最小集）

1. `signals_fast` 返回字段补充：
   - `state`, `current_strength`, `stale`, `expire_reason`, `dedup_remaining_sec`
2. `signal_history` 可增列（建议）：
   - `state`, `is_new`, `market_tag`, `trigger_price`, `expire_reason`
3. 配置项集中化（建议新增）：
   - `SIGNAL_PARAMS = { near_pct, rsi_oversold, breakout_min_pct, half_life_sec, ttl_sec, dedup_sec... }`

### 五、验收标准（本轮必须量化）

1. Pool1 每日信号数较现网下降 20%~40%（去除弱噪声）。
2. 触发后 5 分钟内反向波动超 1% 的“坏信号占比”下降 15%+。
3. `signals_fast` 路径平均响应 < 80ms（池内 200 只）。
4. WS 与 HTTP 降级模式信号一致性 > 99%。

### 六、实施顺序建议（具体）

1. 第1天：`signals.py` 触发逻辑重构 + 单元测试。
2. 第2天：`gm_tick.py` 状态机与去重修复。
3. 第3天：`tick_provider.py` 接口补齐 + `realtime_monitor.py` 适配。
4. 第4天：前端轮询切 `signals_fast` + stale 展示。
5. 第5天：联调压测 + 参数首轮校准 + 回滚预案演练。

### 七、回滚与风险控制

1. 所有新逻辑挂在 `ENABLE_SIGNAL_V2` 特性开关下。
2. 保留旧 `evaluate_pool` 作为 fallback，支持分钟级回切。
3. 先在单池灰度（Pool1 10%样本）观察 2 个交易日，再全量放开。
