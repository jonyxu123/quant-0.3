# 盯盘模块性能运行参数建议（生产可用）

本文给出两套可直接使用的参数模板，适配你当前的单写线程 + 批量事务 + 可选 Redis 缓冲架构。

## 1. 盘中高频（实盘）

适用场景：交易时段实时盯盘、信号触发频繁。

```env
# Writer
REALTIME_DB_WRITER_FLUSH_SEC=0.25
REALTIME_DB_WRITER_MAX_BATCH=1500

# Redis 缓冲（建议开启）
REALTIME_DB_WRITER_REDIS_ENABLED=1
REALTIME_DB_WRITER_REDIS_URL=redis://127.0.0.1:6379/0
REALTIME_DB_WRITER_REDIS_LIST_KEY=quant:realtime:db_write

# Tick/交易流
TICK_PROVIDER=gm
REALTIME_ALLOW_MOCK_FALLBACK=0
```

建议目标：
- `db_writer_queue_size` 常态 < 3000
- `db_writer_batch_tx_fail_total` 保持 0
- `ws_member_cache_fallback_total` 低频增长（偶发可接受）

## 2. 复盘低频（盘后）

适用场景：盘后回放、质量分析、漂移评估，写入压力较低。

```env
# Writer
REALTIME_DB_WRITER_FLUSH_SEC=1.0
REALTIME_DB_WRITER_MAX_BATCH=500

# Redis 缓冲（可关闭）
REALTIME_DB_WRITER_REDIS_ENABLED=0

# Tick/交易流
TICK_PROVIDER=mootdx
REALTIME_ALLOW_MOCK_FALLBACK=0
```

建议目标：
- `db_write_queue_size` 低位稳定
- `db_lock_writer_total` 接近 0
- 质量/漂移线程稳定运行，无持续 error 日志

## 3. 切换与验收

1. 修改 `.env` 后重启后端。  
2. 打开运行时指标接口：`GET /api/realtime/runtime/db_metrics`。  
3. 重点观察：
- `db_writer_flush_sec` 与 `db_writer_max_batch` 是否符合预期
- `db_writer_redis_enabled` / `db_writer_redis_ready` 是否符合配置
- `db_redis_enqueue_fail_total` 与 `db_redis_dequeue_fail_total` 是否持续增长

## 4. 推荐顺序

1. 先用“复盘低频”确认功能稳定。  
2. 再切“盘中高频”并开启 Redis。  
3. 若 Redis 不稳，保持开启但观察 fail 计数；系统会自动回退本地队列，不影响主流程。
