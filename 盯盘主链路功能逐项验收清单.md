# 盯盘主链路功能逐项验收清单

更新时间：2026-04-21  
适用范围：`RealtimeMonitor` 页面 + `backend/api/routers/realtime_monitor.py` + `backend/realtime/*`

## 验收口径
[ ] 通过标准：每项均有“操作结果证据（接口返回/页面截图/关键日志）”。  
[ ] 失败标准：任一主链路阻断（接口 5xx、页面空白、核心字段长期为 0、线程持续报错）。  
[ ] 结论规则：主链路项全部通过才可判定“可用于实盘盯盘”。

## A. 启动与基础连通
[ ] A01 后端启动成功。  
操作：启动后端服务。  
预期：出现 `Application startup complete`，无持续性 `Traceback`。

[ ] A02 盯盘路由注册完整。  
操作：访问 `/api/realtime/pools`。  
预期：HTTP 200，返回池列表（至少包含 Pool1/Pool2）。

[ ] A03 WebSocket 入口可连接。  
操作：前端进入盯盘页或连接 `/api/realtime/ws`。  
预期：连接成功，前端显示“已连接”或进入轮询兜底。

[ ] A04 数据库文件可读写。  
操作：确认 `stock_data.duckdb` 存在且后端无持续“文件被占用无法打开”错误。  
预期：无持续冲突日志，接口可稳定返回。

## B. 行情输入链路（Tick/分钟线/逐笔）
[ ] B01 批量 Tick 可返回。  
操作：调用 `/api/realtime/batch/tick`。  
预期：返回成员股票 `price/open/high/low/pre_close/pct_chg` 字段。

[ ] B02 分钟线可返回。  
操作：调用 `/api/realtime/batch/minute`。  
预期：每只股票返回 1 分钟 K 数据，时间序列连续。

[ ] B03 逐笔成交可返回。  
操作：调用 `/api/realtime/batch/transactions`。  
预期：返回近 N 笔成交，字段含 `time/price/volume/direction`。

[ ] B04 Tick 主字段有效。  
操作：抽样 10 只股票检查。  
预期：`price > 0`，`pre_close > 0`（交易时段），`pct_chg` 非全量 0。

[ ] B05 数据源标记合理。  
操作：检查 `is_mock`。  
预期：实盘时大多数为 `False`；若为 `True` 有明确兜底原因。

## C. 指数链路（页面顶部 6 指数）
[ ] C01 指数数量正确。  
操作：调用 `/api/realtime/indices`。  
预期：固定返回 6 条（上证指数、上证50、沪深300、中证500、深证成指、创业板指）。

[ ] C02 指数中文名正确。  
操作：查看返回 `name` 和前端卡片标题。  
预期：中文名与代码一一对应，无乱码。

[ ] C03 指数价格/涨跌幅有效。  
操作：检查 `price/pre_close/pct_chg`。  
预期：非全 0，`pct_chg` 与 `price-pre_close`方向一致。

[ ] C04 前端指数卡片颜色正确。  
操作：核对涨跌方向。  
预期：上涨红色、下跌绿色，与数值一致。

## D. Pool1（择时监控池）主链路
[ ] D01 Pool1 成员加载正确。  
操作：调用 `/api/realtime/pool/1/members`。  
预期：成员列表、备注、排序可见。

[ ] D02 Pool1 快路径信号可返回。  
操作：调用 `/api/realtime/pool/1/signals_fast`。  
预期：返回 `signals/strength/message/details`。

[ ] D03 Pool1 常规信号可返回。  
操作：调用 `/api/realtime/pool/1/signals`。  
预期：与快路径在结构上兼容，字段完整。

[ ] D04 Pool1 观察态统计可返回。  
操作：调用 `/api/realtime/pool/1/observe_stats`。  
预期：HTTP 200，统计值变化合理。

[ ] D05 Pool1 信号详情字段完整。  
操作：展开信号详情。  
预期：可见触发/确认因子、阈值信息、`resonance_60m_info` 来源信息（缓存/实时/兜底）。

[ ] D06 Pool1 弱信号不过度触发。  
操作：盘中抽样弱条件股票。  
预期：无明显“仅轻微触发即强信号”的误报泛滥。

[ ] D07 Pool1 磁吸过滤行为可解释。  
操作：抽样接近涨跌停个股。  
预期：`blocked/observer_only` 行为与阈值配置一致，不出现双重冲突逻辑。

## E. Pool2（T+0监控池）主链路
[ ] E01 Pool2 成员与信号可拉取。  
操作：调用 `/api/realtime/pool/2/members` 与对应 signals 接口。  
预期：结构完整，返回时间在可接受范围。

[ ] E02 T+0 信号生命周期正常。  
操作：观察同一信号在时间推移中的状态。  
预期：`active -> decaying -> expired` 合理流转，不倒退。

[ ] E03 T+0 质量监控可用。  
操作：调用 `/api/realtime/pool/2/quality_summary`。  
预期：返回 `precision/mfe/mae/churn` 等统计，不持续报错。

[ ] E04 T+0 漂移监控可用。  
操作：调用 `/api/realtime/pool/2/drift_status`。  
预期：返回漂移状态和最近检测时间。

[ ] E05 T+0 打分分层有效。  
操作：检查不同分值样本。  
预期：可区分“可执行/仅观察/不触发”，与策略设定一致。

## F. 前端盯盘页面可用性
[ ] F01 页面可稳定渲染。  
操作：打开盯盘页连续运行 30 分钟。  
预期：无白屏、无频繁重连抖动、无明显内存泄漏。

[ ] F02 股票卡片价格与涨跌幅正确。  
操作：随机抽样 10 只股票与接口对比。  
预期：价格、涨跌幅、颜色方向一致；不再长期显示 0%。

[ ] F03 信号详情可读。  
操作：展开卡片信号详情。  
预期：关键信息齐全，字段命名一致，无乱码关键文案。

[ ] F04 排序/筛选可用。  
操作：按涨跌幅、强度切换排序。  
预期：排序结果与数据一致，无明显错位。

[ ] F05 轮询兜底可用。  
操作：断开 WS 后观察页面。  
预期：自动进入轮询并继续更新数据。

## G. 持久化与回放
[ ] G01 Tick 持久化线程正常。  
操作：观察启动日志与运行日志。  
预期：无持续 `tick persistence` 异常。

[ ] G02 信号历史可查询。  
操作：调用 `/api/realtime/pool/{pool_id}/signal_history`。  
预期：返回记录可用于复盘，时间字段正确。

[ ] G03 数据写入不阻塞推送。  
操作：高频时段观察 WS 推送与接口延迟。  
预期：无明显“写入导致前端卡死/推送中断”。

## H. 稳定性与告警
[ ] H01 无持续性 NameError/ImportError。  
操作：观察 15 分钟日志。  
预期：不再出现 `_main_recent_txns`、`TXN_ANALYZE_COUNT`、`TICK_PROVIDER` 未定义。

[ ] H02 无持续 DuckDB 文件锁冲突。  
操作：观察日志中 IO Error。  
预期：不出现持续“另一个程序正在使用此文件”刷屏。

[ ] H03 错误可降级。  
操作：人为断开数据源。  
预期：系统降级到兜底数据或观察态，不崩溃。

## I. 关键回归项（本轮必须通过）
[ ] I01 6 个指数完整显示且中文名正确。  
[ ] I02 股票卡片涨跌颜色规则正确（涨红跌绿）。  
[ ] I03 股票卡片涨跌幅不再全量 0%。  
[ ] I04 Pool1 `resonance_60m_info` 在信号详情可视化。  
[ ] I05 Pool1 `observe_stats` 稳定返回。  
[ ] I06 Pool2 质量监控接口稳定返回。  
[ ] I07 启动后 15 分钟内无高频致命错误刷屏。

## 验收结论
[ ] 结论：主链路通过，可继续做策略优化迭代。  
[ ] 结论：主链路未通过，需先修复阻断项再进入策略优化。
